Le variabili

Servono per memorizzare dei valori

Ogni variabile è identificata da un nome che segue alcune regole:

  • niente spazi, niente trattini -, niente caratteri speciali (? , )
  • deve iniziare con una lettera, sono ammessi anche underscore e dollaro _, $
  • sono case sensitive
  • non può iniziare con un numero
  • non deve mai coincidere con le keywords di javascript (if, else, switch, return, true, false)

Dichiarazione implicita delle variabili

Non è obbligatorio dichiarare le variabili, cioè fare un'operazione esplicita di creazione
quando c'è l'identicatore = l'engine crea implicitamente la variabile se non esiste già

Strict mode

E' un'opzione di controllo del codice, EcmaScript 5 -> se viene dichiarato lo strict mode javascript viene eseguito osservando delle regole di scrittura del codice più strette -> es. ci arriva segnalazione di errore se non dichiariamo le variabili Nell'esecuzione "normale" javascript ha delle regole più permissive che se dichiariamo lo strict mode ci obbligano a uno stile di programmazione più strutturato

Var, let, const

var è la prima keyword usata da javascript per usare una variabile, var è accessibile in qualunque parte dello script (variabile globale)

let introdotta da Ecmascript6 è accessibile soltanto in un determinato ambito (scope) che è delimitato da un blocco di codice

Un blocco di codice javascript è una serie di istruzioni delimitate da { .. blocco di codie }

const

Introdotte da EcmaScript 6 -> istruisce l'engine che il valore assegnato non è modificabile
